# 자동 로그인 설정 가이드 (디자인 작업용)

## 현재 상태
`.env.local` 파일에 `NEXT_PUBLIC_USE_MOCK_DATA=true`가 설정되어 있습니다.

## 로그인 유지 방법

### 옵션 1: 브라우저에서 직접 로그인
1. http://localhost:3000/sign-in 접속
2. 아무 이메일/비밀번호 입력 (예: test@test.com / 123456)
3. 로그인 후 브라우저 개발자 도구(F12) → Application → Local Storage
4. `amiko_session` 키 확인 - 이것이 로그인 상태 유지

### 옵션 2: LocalStorage에 직접 세션 추가
브라우저 콘솔(F12)에서 실행:
```javascript
localStorage.setItem('amiko_session', JSON.stringify({
  user: {
    id: 'test-user',
    email: 'design@test.com',
    user_metadata: { name: '디자인 테스트', full_name: '디자인 테스트' }
  },
  expires_at: Math.floor(Date.now() / 1000) + 86400 // 24시간 후
}));
localStorage.setItem('amiko_token', 'dummy-token');
location.reload();
```

### 옵션 3: 로그인 페이지 우회
현재 `isAdmin = true`로 설정되어 있어 모든 UI가 표시됩니다.
로그인 없이도 대부분의 디자인을 확인할 수 있습니다.

## 참고
- 디자인 작업 완료 후 이 설정들은 제거하거나 실제 인증으로 교체해야 합니다
- 실제 배포 시에는 반드시 실제 Supabase 인증을 사용해야 합니다
