# 브랜치별 개발 서버 실행 가이드

## ✅ 네, 브랜치 전환 후에도 개발 서버 실행 가능합니다!

### 🔄 브랜치와 개발 서버의 관계

**브랜치는 코드의 다른 버전을 관리하는 것이고, 개발 서버는 그 코드를 실행하는 것입니다.**

- ✅ 각 브랜치에서 독립적으로 개발 서버 실행 가능
- ✅ 브랜치 전환 후 개발 서버 재시작 권장
- ✅ 여러 브랜치를 번갈아가며 작업 가능

### 📋 작업 흐름

#### 시나리오 1: main 브랜치에서 개발 서버 실행
```bash
# 1. main 브랜치로 전환
git checkout main

# 2. 개발 서버 실행
npm run dev

# → http://localhost:3000 에서 main 브랜치 코드 실행
```

#### 시나리오 2: 영상통화 브랜치에서 개발 서버 실행
```bash
# 1. 영상통화 브랜치로 전환
git checkout feature/video-call-development

# 2. 개발 서버 실행 (또는 재시작)
npm run dev

# → http://localhost:3000 에서 영상통화 브랜치 코드 실행
```

### ⚠️ 주의사항

1. **브랜치 전환 시 개발 서버 재시작 권장**
   - 브랜치를 전환하면 파일 내용이 바뀔 수 있음
   - 개발 서버가 변경사항을 자동 감지하지만, 재시작하는 것이 안전

2. **포트 충돌**
   - 같은 포트(3000)를 사용하므로 한 번에 하나의 브랜치만 실행 가능
   - 다른 브랜치로 전환하려면 현재 서버를 중지하고 새로 시작

3. **의존성 변경**
   - 브랜치마다 `package.json`이 다를 수 있음
   - 브랜치 전환 후 `npm install` 실행 권장

### 🎯 실전 예시

#### main 브랜치에서 작업 중
```bash
# 현재: main 브랜치
# 개발 서버 실행 중: npm run dev
# → http://localhost:3000 에서 main 브랜치 실행
```

#### 영상통화 브랜치로 전환하여 작업
```bash
# 1. 개발 서버 중지 (Ctrl+C)

# 2. 브랜치 전환
git checkout feature/video-call-development

# 3. 개발 서버 재시작
npm run dev

# → http://localhost:3000 에서 영상통화 브랜치 실행
```

### 💡 팁

1. **터미널 탭 분리**
   - 터미널 1: main 브랜치용 개발 서버
   - 터미널 2: 영상통화 브랜치용 개발 서버
   - → 포트를 다르게 설정하면 동시 실행 가능 (예: 3000, 3001)

2. **환경 변수 확인**
   - 브랜치마다 다른 환경 변수가 필요할 수 있음
   - `.env` 파일 확인

3. **데이터베이스**
   - 같은 데이터베이스를 사용하므로 주의
   - 테스트 데이터 사용 권장

### 🚀 현재 상태

- **현재 브랜치**: `feature/video-call-development`
- **개발 서버 실행**: `npm run dev`
- **접속 주소**: `http://localhost:3000`

✅ **개발 서버 실행 = 그 코드 실행**
✅ **각 브랜치에서 독립적으로 개발 서버 실행 가능**
